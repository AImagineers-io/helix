# P87: Launch Preparation

## 1. Objective

- **What**: Documentation, demo instance, and launch checklist for production readiness
- **Why**: P85-P86 deliver a working system; this phase makes it deployable and demonstrable
- **Scope**:
  - Included: Docs updates, .env.example, demo setup, launch checklist
  - Excluded: New features, architectural changes, client-specific customization

## 2. Implementation Tasks

- [ ] Environment template (1 file, ~50 LOC)
  - Tests: tests/integration/test_env_template.py
  - Files: .env.example
  - Include: All required variables with descriptions, safe defaults
  - Depends on: P85 complete

- [ ] Deployment guide update (1 file, ~200 LOC)
  - Tests: (manual verification)
  - Files: docs/deployment-guide.md
  - Sections: Prerequisites, config, docker-compose, cloudflare tunnel, verification
  - Depends on: P87.1

- [ ] Client onboarding checklist (1 file, ~100 LOC)
  - Tests: (manual verification)
  - Files: docs/client-onboarding.md
  - Sections: QA prep, branding, Messenger setup, go-live steps
  - Depends on: P87.2

- [ ] Troubleshooting guide (1 file, ~100 LOC)
  - Tests: (manual verification)
  - Files: docs/troubleshooting.md
  - Cover: Common errors, log locations, health checks, recovery steps
  - Depends on: P87.2

- [ ] Demo QA dataset (1 file, ~100 LOC)
  - Tests: tests/integration/test_demo_data.py
  - Files: data/demo-qa-pairs.json
  - Content: 50 generic QA pairs (not client-specific)
  - Depends on: None

- [ ] Demo instance script (2 files, ~80 LOC)
  - Tests: tests/integration/test_demo_script.py
  - Files: scripts/setup_demo.py, scripts/reset_demo.py
  - Features: Load demo data, apply demo branding, auto-reset capability
  - Depends on: P87.5

- [ ] Launch checklist (1 file, ~60 LOC)
  - Tests: (manual verification)
  - Files: docs/launch-checklist.md
  - Items: Config verified, tests pass, migrations applied, monitoring ready, rollback plan
  - Depends on: P87.2

## 3. Success Criteria

- [ ] .env.example documents all required environment variables
- [ ] Deployment guide enables fresh deployment from scratch
- [ ] Client onboarding checklist covers all handoff items
- [ ] Troubleshooting guide addresses top 10 anticipated issues
- [ ] Demo instance loads and responds within 2 seconds
- [ ] Demo reset script returns instance to clean state
- [ ] Launch checklist has no unchecked items before deploy
- [ ] All documentation reviewed for accuracy
